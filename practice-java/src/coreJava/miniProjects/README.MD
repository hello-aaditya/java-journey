# 🗂️ Mini Projects — Core Java

This folder contains a growing collection of small, self-contained Java projects built as part of the **Core Java learning journey**. Each project is intentionally scoped to reinforce a specific cluster of Java concepts learned up to that point — no frameworks, no databases, just pure Java.

---

## 📌 Purpose

Mini-projects bridge the gap between isolated practice exercises and real-world applications. Rather than writing snippets, you build a small but complete program that has a clear business purpose, realistic requirements, and structured code.

Each project:
- Has a well-defined problem statement (see its own `README.md`)
- Is solved using **only the Java concepts covered so far**
- Encourages thinking about design, not just syntax
- Grows slightly more complex as the journey progresses

## 💡 How to Approach Each Project

1. **Read the README first** — understand the business problem before writing any code.
2. **Design before you type** — think about what classes, methods, and variables you need.
3. **Build incrementally** — get one piece working, then add the next.
4. **Review against requirements** — check every bullet point in the spec once you're done.
